# §10 THE METHODOLOGY (How To Approach Building)

## Game-Dev Wisdom That Emerged:

1. **Prototype core loop first** — Not whole world. The minimum that creates the feeling.

2. **Feel before feature** — Does it produce "I CAN SEE"? If not, iterate before adding more.

3. **Find the fun first** — If using it doesn't feel like playing a game, the design is wrong.

4. **Players do unexpected things** — Design for emergence, not rigid paths.

5. **Paper prototype before digital** — Can we test the concept with existing tools before building custom?

## What This Means For You:

If you're building the artifact:
- Get the data persistence SOLID first (data loss = catastrophic trust destruction)
- Get the Actual filter working (panic button must work)
- Get basic DAG rendering working (can see tasks and dependencies)
- THEN add visual polish

Don't build the beautiful orbital macro view before the core micro loop works.
